## Issue P1-01: Keep dialogue report path patterns portable

Title:
`P1: keep dialogue_governance_contract report patterns repo-portable (no absolute runtime root rewrites)`

Severity:
`P1`

Problem:
`create_identity_pack.py` currently injects relative dialogue report patterns in scaffold skeleton, but later runtime-root rewriting can convert path-like fields to host-specific absolute paths. This weakens portability across machines and environments.

Why it matters:

1. Protocol contracts should remain machine-agnostic in stored task definitions.
2. Absolute paths in contract patterns increase drift risk in shared/runtime migration.

Scope:

1. `scripts/create_identity_pack.py`
2. `identity/*/CURRENT_TASK.json` samples generated by scaffold (validation target only)

Required change:

1. Preserve `dialogue_governance_contract` pattern fields as relative contract paths:
   - `report_path_pattern`
   - `dialogue_content_report_path_pattern`
   - `dialogue_cross_validation_report_path_pattern`
   - `dialogue_result_support_report_path_pattern`
2. Runtime resolution should happen at validator runtime, not at scaffold write time.

Non-goals:

1. No change to dialogue validator semantics (`required=false`, `warn/enforce`).
2. No change to report discovery fallback order unless separately reviewed.

Acceptance commands:

```bash
python3 scripts/create_identity_pack.py --id tmp-portable-min --title "Tmp" --description "Tmp" --pack-root /tmp/ip-portable/min --profile minimal --skip-bootstrap-check
python3 scripts/create_identity_pack.py --id tmp-portable-full --title "Tmp" --description "Tmp" --pack-root /tmp/ip-portable/full --profile full-contract --skip-bootstrap-check
python3 - <<'PY'
import json,glob,os,sys
paths = glob.glob('/tmp/ip-portable/**/CURRENT_TASK.json', recursive=True)
ok = True
for p in paths:
    d = json.load(open(p, encoding='utf-8'))
    c = d.get('dialogue_governance_contract', {})
    for k in ('report_path_pattern','dialogue_content_report_path_pattern','dialogue_cross_validation_report_path_pattern','dialogue_result_support_report_path_pattern'):
        v = str(c.get(k,''))
        if '/Users/' in v:
            ok = False
            print('[FAIL]', p, k, v)
if not ok:
    sys.exit(1)
print('[OK] dialogue report patterns remain portable/relative')
PY
```

Definition of done:

1. New scaffold outputs keep portable relative patterns.
2. No regression in `scripts/validate_identity_dialogue_*` execution.
3. `scripts/docs_command_contract_check.py` remains PASS.

## Issue P1-03: Standardize local protocol audit toolchain preflight

Title:
`P1: add deterministic local preflight for actionlint/ast-grep/gitleaks parity with CI`

Severity:
`P1`

Problem:
CI has required gates for `actionlint`, `ast-grep`, `gitleaks`, but local review environments may miss binaries, causing non-reproducible local audits.

Why it matters:

1. Faster reviewer feedback loops.
2. Lower CI-only surprise rate.

Scope:

1. New script (recommended): `scripts/preflight_protocol_audit_env.sh`
2. Optional docs update: `README.md` local audit section

Required change:

1. Preflight script checks presence/version of:
   - `actionlint`
   - `sg` (ast-grep)
   - `gitleaks`
2. Exit non-zero with actionable install hints if missing.
3. Keep script non-destructive (check-only).

Non-goals:

1. Do not auto-install tools inside protocol validation scripts.
2. Do not change CI gate semantics.

Acceptance commands:

```bash
bash scripts/preflight_protocol_audit_env.sh
python3 scripts/validate_protocol_ssot_source.py
python3 scripts/validate_protocol_handoff_coupling.py --base HEAD~1 --head HEAD
```

Definition of done:

1. Local preflight gives deterministic pass/fail.
2. Reviewers can reproduce CI toolchain prerequisites before running full checks.
3. Existing required-gates workflow remains unchanged.
